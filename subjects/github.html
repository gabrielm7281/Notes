General Knowledge

Github is the central place for every programming project to originally start off in. It is very useful for viewing code and hosting websites, but it is a little limited in its function. The main place to code and get previews of your work is your IDE. Since Github cant run all kinds of languages, you need to have other ways to preview things and that is why you use an IDE to allow for these limitations. From the IDE, you can work on your project even without wifi if it is a local project. That means if it is downloaded and installed onto your device, you can run/edit the program anywhere you want. However you do need wifi to actually push the project onto Github, which is where it is publicly availible, and where you can put it onto a website. You can push (move from computer to Github), pull (from Github to IDE) and fork (copy someone elses project) into the IDE. You can push and pull from a central place and Github merges the two saves together once everything is loaded.

Merges: A git merge is when you are working on some code in a local IDE but another piece of code on Github is different. When these two commits are eventually combined, they will automatically merge together but ONLY IF the lines of code are 2 lines apart and don't conflict with each other.

Merge Conflict: A merge conflict happens when the above 2 conditions aren't met. Your system might freak out but it isnt very difficult to overcome. To fix a merge conflict you have to choose which one of the lines you want to keep in your repo and re commit and push the code so that the issue is fixed. The SEP IDE might prompt you to resolve the conflict in its merge editor. It is a little more confusing if you use the merge editor but it is still fairly easy to understand
